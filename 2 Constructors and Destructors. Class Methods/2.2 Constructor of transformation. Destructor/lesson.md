# 2.2 Конструктор преобразования. Деструктор

## Конструктор преобразования

Конструктор с одним параметром, тип которого отличается от типа класса, называется **конструктором преобразования**. Он позволяет компилятору неявно преобразовывать значение одного типа в объект класса.

Рассмотрим класс для представления комплексных чисел:

```cpp
class Complex {
public:
    Complex() {}
    Complex(double real) : re{real}, im{0.0} {}
    Complex(double real, double imag) : re{real}, im{imag} {}

    void get_data(double& re, double& im) {
        re = this->re;
        im = this->im;
    }

private:
    double re{0.0};
    double im{0.0};
};
```

Объекты класса `Complex` можно создавать несколькими способами:

```cpp
int main() {
    Complex c1;              // конструктор по умолчанию
    Complex c2(0.5);         // конструктор с одним параметром
    Complex c3(-5.4, 7.8);   // конструктор с двумя параметрами

    return 0;
}
```

Фигурные скобки являются полным аналогом:

```cpp
Complex c1{};
Complex c2{0.5};
Complex c3{-5.4, 7.8};
```

Также допустим синтаксис инициализации через `=`:

```cpp
Complex c1;
Complex c2 = 0.5;           // неявный вызов конструктора преобразования
Complex c3 = {-5.4, 7.8};
```

При создании `c2` справа находится значение типа `double`, а слева формируется объект типа `Complex`. Конструктор с одним параметром выполняет это преобразование. Именно поэтому такие конструкторы называются **конструкторами преобразования**.

### Неявное преобразование при вызове методов

Конструктор преобразования работает не только при инициализации, но и при передаче аргументов в методы. Рассмотрим метод сложения комплексных чисел:

```cpp
const Complex& add(const Complex& other) {
    this->re += other.re;
    this->im += other.im;
    return *this;
}
```

Стандартный вызов:

```cpp
c2.add(c3);  // передаётся объект Complex
```

Но допустим и такой вызов:

```cpp
c2.add(4.3);  // передаётся double → неявное преобразование в Complex
```

Компилятор обнаруживает, что метод ожидает `const Complex&`, а получает `double`. Он находит подходящий конструктор преобразования `Complex(double)`, создаёт временный объект и передаёт его в метод.

> **Важно:** неявное преобразование работает только для одного шага. Компилятор не выполняет цепочку из нескольких неявных преобразований.

## Ключевое слово explicit

Если неявное преобразование типов нежелательно, перед конструктором преобразования указывается ключевое слово `explicit` (начиная с C++11):

```cpp
explicit Complex(double real) : re{real}, im{0.0} {}
```

После этого:

```cpp
Complex c2 = 0.5;        // ошибка: неявный вызов запрещён
Complex c2(0.5);         // ok: явный вызов
Complex c2{0.5};         // ok: явный вызов
```

Аналогично при вызове методов:

```cpp
c2.add(4.3);             // ошибка: неявное преобразование запрещено
c2.add(Complex(4.3));    // ok: явное создание объекта
```

### Когда следует использовать explicit

Рекомендуется ставить `explicit` на все конструкторы с одним параметром, если нет явной причины разрешать неявное преобразование. Это предотвращает случайные конверсии и делает код более предсказуемым:

```cpp
class FileHandle {
public:
    explicit FileHandle(int fd) : fd{fd} {}  // без explicit: FileHandle f = 42; компилируется!

private:
    int fd;
};
```

> **Начиная с C++11** ключевое слово `explicit` можно использовать и для конструкторов с несколькими параметрами (запрещает инициализацию через `= {}`), а также для операторов преобразования (`explicit operator bool()`).

## Деструктор класса

**Деструктор** - специальный метод класса, который вызывается непосредственно перед уничтожением объекта (освобождением занимаемой им памяти).

Свойства деструктора:

- имя совпадает с именем класса, но с префиксом `~` (тильда);
- не возвращает значений (возвращаемый тип **не указывается**);
- **не имеет параметров**;
- в классе может быть **только один** деструктор;
- назначение - **освобождение ресурсов**, захваченных объектом.

### Пример: класс с динамическим массивом

Рассмотрим класс для представления точки в N-мерном пространстве:

```cpp
class PointND {
public:
    PointND() {}

    PointND(unsigned sz) : total{sz} {
        coords = new int[total]{0};
    }

    PointND(int* cr, unsigned len) : total{len} {
        coords = new int[total];
        set_coords(cr, len);
    }

    ~PointND() {
        delete[] coords;
    }

    unsigned get_total() { 
        return total; 
    }

    const int* get_coords() { 
        return coords; 
    }

    void set_coords(int* cr, unsigned len) {
        for (unsigned i = 0; i < total; ++i) {
            coords[i] = (i < len) ? cr[i] : 0;
        }
    }

private:
    unsigned total{0};
    int* coords{nullptr};
};
```

Без деструктора при уничтожении объекта память, выделенная через `new`, не освобождалась бы - возникала бы **утечка памяти**:

```cpp
int main() {
    PointND pt(5);  // без деструктора - утечка памяти

    return 0;
}
```

Деструктор `~PointND()` вызывается автоматически:

- для локальных объектов - при выходе из области видимости;
- для объектов в динамической памяти - при вызове `delete`.

```cpp
int main() {
    PointND pt(5);              // деструктор вызовется при выходе из main

    PointND* ptr = new PointND(3);
    delete ptr;                  // деструктор вызовется здесь

    return 0;
}
```

### Порядок вызова деструкторов

Деструкторы локальных объектов вызываются в **обратном порядке** по отношению к их созданию:

```cpp
{
    PointND a(2);   // создаётся первым
    PointND b(3);   // создаётся вторым
}   // ~b(), затем ~a()
```

### Правило трёх (Rule of Three)

Если класс управляет ресурсами (динамическая память, файловые дескрипторы и т.д.) и определяет деструктор, то, как правило, он также нуждается в пользовательских:

- конструкторе копирования;
- операторе присваивания копированием.

Это связано с тем, что компилятор по умолчанию генерирует **поверхностные копии** (побитовое копирование указателей), что приводит к двойному освобождению одной и той же памяти. Подробнее - в следующих уроках.
